Holistic 3D

3D Model Synthesis from a 360 degree view panoramic image.

준비사항

- Input: image file

  • File type: .png
  • Size: Under 2MB
API Document Upload : Make a creation request through pcd 3d model creation request, and receive a request key to know the processing status.

① Request

  • Method : POST
  • URL : https://api.maum.ai/holistic3d/upload

② Request parameters

Name Description type
apiId Unique API ID. Request from is required for the ID. string
apiKey Unique API key. Request from is required for the key. string
image type:file (.png) image file file

③ Request example

curl --location --request POST 'https://api.maum.ai/holistic3d/upload' \
--header 'Content-Type: multipart/form-data' \
--form 'apiId= Own API Id'\
--form 'apiKey= Own API Key'\
--form 'image= Input image file'\

④ Response parameters

Response
Name Description type
message API operation object
payload result object
message: API operation
Name Description type
message String describing the request processing status (Success/Fail) string
status Status code for request processing status (0: Success) int
payload: result
Name Description type
status follow status code of mmsr
status(1): message -> file_key
status(2): upload error 		int
fileKey Unique Key for Request Identification string
msg msg for request string

⑤ Response example

{
	"message": {
		"message": "Success",
		"status": 0
	},
	"payload": {
		"status": "1"
		"fileKey": "369f5394-dca7-4356-813e-60ae9a74be80"
		"msg": "success"
	}
}
Check process : Check the processing status of the request.

① Request

  • Method : POST
  • URL : https://api.maum.ai/holistic3d/check:process

② Request parameters

Name Description type
apiId Unique API ID. Request from is required for the ID. string
apiKey Unique API key. Request from is required for the key. string
fileKey Unique Key for Request Identification string

③ Request example

curl --location --request POST 'https://api.maum.ai/holistic3d/check:process' \
--header 'Content-Type: application/json' \
--form 'apiId= Own API Id'\
--form 'apiKey= Own API Key'\
--form 'fileKey= Unique Key for Request Identification' \

④ Response parameters

Response
Name Description type
message API operation object
payload result object
message: API operation
Name Description type
message String describing the request processing status (Success/Fail) string
status Status code for request processing status (0: Success) int
payload: result
Name Description type
status NOT_YET = 6;
PROCESSING = 7;
DONE = 8;
WRONG_KEY = 10;
ERROR = 11;
DELETED = 9; 		int
message Phrases according to status code string

⑤ Response example

{
	"message": {
		"message": "Success",
		"status": 0
	},
	"payload": {
		"status": 8,
		"message": "Process Completed(can download)"
	}
}
Download: Download the processed 3D .pcd file.

① Request

  • Method : POST
  • URL : https://api.maum.ai/holistic3d/download

② Request parameters

Name Description type
fileKey Unique Key for Request Identification string

③ Request example

curl --location --request GET
'https://api.maum.ai/holistic3d/download?fileKey= Unique Key for Request Identification'

④ Response example

Content-Type: application/octet-stream
(.pcd File Download)
